\#1. 로봇 스킬 마켓



* 문제 

1\. AI 로봇의 여러 태스크를 동시에 학습시키면 충돌이 나기 쉬움. 

2\. 하나의 태스크를 완전한 동작으로 학습시키는데 연 단위의 시간이 걸림.



* 제안 내용

'계단 걷기', '상자 쌓기' 등 특정 동작을 완벽하게 수행하는 AI 모델(스킬)을 거래하는 플랫폼. 

스마트폰이 앱을 통해 기능을 확장하듯, 로봇 하드웨어에 관계없이 특정 동작(Skill)을 다운로드하여 실행하게 하는 모델.

테스크마다 독립된 브레인으로, 각 행동에 대한 브레인을 사서 적용 가능한 플랫폼.

(\*로봇 제조사들이 공통으로 사용할 수 있는 '표준 API 레이어'를 제공)

핵심 : 스킬을 각 바이어의 로봇에 맞게 Sim-2-Real 최적화.



* 필요 기술
1. 스킬 거래 플랫폼
2. 구성 파일 매핑 : 구매한 스킬 데이터안에 담긴 '행동 의도'를 구매자 로봇의 하드웨어 프로필을 읽어서 IK를 재계산.
예) 구매자 로봇 : 7축 관절,  스킬 정의 로봇 : 6축 관절
3. 파인튜닝 : 스킬을 처음 적용할 때, 로봇이 실행해보며 현재 모터 상태를 체크하여 스킬에서 필요 데이터의 수치를 미세 조정
4. 제조사마다 다른 '로봇 통신 규격'을 통합을  'ROS2(Robot Operating System)' 같은 오픈소스 표준을 활용

&#x09;

* 서비스 수혜자 : 

\- 기업: 로봇 하드웨어만 사고 필요한 기능을 여기서 다운받아 개발의 기반을 빠르게 다지고 시간을 아낄 수 있음.

\- AI 개발자 : 해당 스킬의 학습 방법 등을 공유할 수 있어, J곡선의 로보틱스 성능 향상을 기대해볼 수 있음. 



* 근거 : 아크 레이더스의 개발자분의 발표 기사글 https://www.gamevu.co.kr/news/articleView.html?idxno=56524



하나의 뉴럴 네트워크에 '전진하기', '정지하기', '짓밟기' 같은 여러 태스크를 동시에 학습시키면 태스크끼리 간섭한다. 예를 들어 '전진'과 '정지'를 동시에 가진 로봇은 '언제 정지 명령이 올지 모른다'는 공포에 떠는 것처럼 어정쩡하게만 이동하게 되어 버린 것이다.



\-> 태스크마다 독립된 '브레인'을 준비하고, 게임 엔진 측의 행동(비헤이비어) 트리(적 AI의 행동 패턴을 '만약 ○○라면 △△한다'는 트리 형태의 플로차트로 관리하는 구조)로 상황에 따라 브레인을 전환한 것이다. '전진 브레인'은 정지라는 개념을 전혀 모르므로 전력으로 질주한다. 정지가 필요한 순간에 '정지 브레인'으로 전환되어 제대로 멈추는 식이다.













\#2. 로봇 간병인 서비스



* 문제
1. 우리나라 국민 5명 중 1명이 고령인구인 '초고령사회'에 올해 첫 진입한 사실이 통계로 확인 (https://www.nocutnews.co.kr/news/6407362 - 2025 고령자 통계)

2\.    간병비 월 200\~300만원대로, 개인 부담이 높다는 통계

3\.     기존 '인공지능(AI) 스마트 간병기기'가 있지만, 보고 및 알림 방식이라 물리적 보조자 역할을 수행하지 못하여 한계가 많음.



* 제안 내용 

로봇이 환자의 거동을 보조하고, AI가 실시간으로 보행 패턴과 안색을 분석해 이상 징후 발생 시 병원과 보호자에게 즉시 알림을 보냄.

\+환자가 약을 먹었는지 등 확인하고, 재활 운동시 동작을 교정해줌.
